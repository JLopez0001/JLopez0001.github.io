Introduction

Welcome to RoadRunner, a mobile fitness social media app that elevates your running experience. Utilizing GPS technology, RoadRunner tracks your run time, pace, and distance, allowing you to share and connect over your fitness achievements. With RoadRunner, you can follow friends, like, and comment on each other's runs, creating a supportive and interactive community of runners.

Idea behind RoadRunner

UI Experience

Getting Started

In order to use RoadRunner that user will need to create an account and login.

Feed Page

On the Feed Page, users can view the running activities of the people they follow. Each post displays the distance traveled, pace, and time of the run, allowing users to stay connected and motivated by their community's achievements.

Create A Comment and Like

What's a social media app without the ability to engage with content? RoadRunner allows you to like and comment on the runs posted by your friends, fostering a dynamic and supportive community where every milestone is celebrated and every challenge is shared.

Run

When users are ready to hit the pavement, RoadRunner utilizes the Mapbox API to fetch their current coordinates. As they run, the app draws a line in real time on the map, visually tracking the route and distance covered. This feature enhances the running experience, allowing users to see their progress as they move.

Challenges

Major Challenge on this Project

A significant challenge in developing RoadRunner was mastering geolocation technology, particularly in ensuring real-time updates of user coordinates. As users moved, their locations were stored in an array, intended to be used to draw their running path. However, I encountered issues with duplicate coordinates, which interfered with the path visualization. To resolve this, I implemented a filter mechanism to remove any duplicates, ensuring the line is accurately drawn from one coordinate to the next.
